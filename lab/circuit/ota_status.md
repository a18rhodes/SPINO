### Attribution (Probe 1 — 2026-05-11): COMPLETE

Root cause identified: **M4 PFET output mirror** is the dominant error source.
- M4 max|ΔI|: 15.4 µA (L=0.40), 9.2 µA (L=0.50)
- Next highest: M3 5.6 µA, M2 4.8 µA (L=0.40)
- M5 (tail): 2.1 µA (L=0.40), 0.5 µA (L=0.50) — negligible

Mechanism: when n_out slews near VDD, M4 exits saturation (Vds → 0). PFET
training data underrepresents this triode-boundary regime, causing ~15 µA
current overestimate → ~70 mV voltage offset at the saturation plateau.

The slew-time failure at L=0.40 is a downstream consequence: the 10–90%
threshold shifts by ~3.5 ns due to the plateau offset.

## Current status (2026-05-11): RUNTIME OPTIMIZATION

Phase 3 is complete. JVP-GMRES implementation lives in OtaTransientSolver
as an opt-in (use_gmres=True) alternative to the dense Jacobian path.
Default remains use_gmres=False: on the OTA problem sizes the GPU-batched
dense backward outperforms sequential scipy-GMRES JVP matvecs by roughly
7x, so the dense route is the production default.

Initial GPU (dense Jacobian, use_gmres=False): ~65 s per run.
GMRES on GPU: slower than the dense path at OTA scale; retained for
larger circuits where dense assembly stops fitting on-device.

## PFET triode fine-tune (2026-05-15): PARTIAL CLOSURE

**Goal:** Close the OTA L=0.40 max|ΔV| gate by adding PFET training samples
covering the Vsd→0 triode regime that attribution localized to M4.

**Augmentation set.** 2K samples generated with the existing `sky130_pmos`
strategy, restricted gate/drain ranges (Vg=0.2–0.8 V, Vd=1.5–1.8 V; source
and bulk fixed at VDD), PWL waveform mode, `t_end=1 µs`, 2048 timesteps.
This puts Vsg ∈ [1.0, 1.6] V (strong inversion) with Vsd ∈ [0.0, 0.3] V
(triode boundary). Output: `datasets/sky130_pmos_triode_aug_2k.h5`.

**Merged dataset.** `merge_geometry_bins` over the existing 44K production
PFET dataset + the 2K triode aug → `datasets/sky130_pmos_50k_triode.h5`
(46K samples, shuffled).

**Fine-tune.** Frozen-backbone, FiLM-conditioning layers only (176K of 2.33M
params trainable, 7.5%), 50 epochs, LR=1e-4. Final LpLoss 0.147 (production
baseline 0.037 on smaller dataset). 35 min wall time.

Checkpoint: `spino/models/mosfet/pfet/pmos_exp07_triode_finetune_KG1HfPbJ.pt`

**Device-level eval (sky130_pmos, production sizing):**

| Metric | Production | Fine-tune | Δ |
|---|---|---|---|
| SPICE Transfer R² | 0.9965 | 0.9957 | −0.0008 |
| SPICE Output MALE | 60 µA | 21.7 µA | **−64%** |
| tiny Sweep R² | 0.9973 | 0.9982 | +0.0009 |
| medium Sweep R² | 0.9957 | 0.9976 | +0.0019 |
| xlarge Sweep R² | 0.9942 | 0.9592 | **−0.0350** (xlarge regressed) |

**OTA composition (production sizing, L=0.40, W_diff=W_mirror=8 µm):**

| | Production PFET | Triode fine-tune |
|---|---|---|
| max\|ΔV\| | 68.7 mV | 61.0 mV (−11%) |
| M4 peak \|ΔI\| (triode target) | 15.4 µA | 12.0 µA (−22%) |
| M3 peak \|ΔI\| (non-triode PFET) | 5.6 µA | 6.0 µA (+8%, regressed) |
| Pearson r | 0.9997 | 0.9997 |

**Interpretation.** M4 reduced 22 % — real, attribution-traceable improvement
on the targeted device. Composition error reduction (11 %) smaller than
M4 device improvement because Newton coupling distributes the residual error
across all three KCL nodes. M3 regressed slightly — FiLM-only adaptation
over-specializes to the triode regime. **30 mV gate still fails by 31 mV.**

The residual gap is no longer purely a coverage problem. Augmenting triode
data narrowed but did not close it. Candidate next steps if the gate must
close: (a) larger triode augmentation (5–10K samples), (b) full-backbone
fine-tune at very tight LR with seed sweep to avoid the xlarge regression
seen here, (c) geometry-stratified retrain weighted by the OTA's actual
operating bin. **Deferred to future work.**

Production PFET checkpoint unchanged. Fine-tune checkpoint archived
alongside attribution artefacts as a documented partial-closure result.
